# 项目协作规则

## 项目目标

- 这个项目用于：开发一个 Obsidian 插件，在笔记侧边栏中与 AI 进行**多轮对话**，AI 能记住对话历史，并根据当前笔记内容辅助编辑（总结、续写、翻译、问答等）。基本功能照抄claudian的："F:\obsidian-ollama\claudian"，claudian是开源的MIT协议，可以照抄。不用闭门造车，不要重复造轮子，本项目的主要目标是开发一款和claudian基本一致的APP，不同点在于不需要那么复杂的功能，暂时不需要支持claude和codex等国外模型。**重点是原生支持deepseek，小米mimo2.5，ollama的gemma4模型**。
- 有户界面和设置界面必须全部使用中文，除非是代码标识符、命令原文。
- 主要用户和场景：Obsidian 用户，尤其希望使用**本地 Ollama 模型**或国内 API（DeepSeek、mimo-v2.5 等），在离线或内网环境下稳定工作。由于本地模型容易“丢失上下文”，**多轮对话是保证可用性的核心**。
- 当前阶段：从零开始搭建**最小可用原型**，但必须包含：
  - 侧边栏对话界面（显示完整对话历史）
  - 多轮对话的上下文管理（自动携带历史消息）
  - 支持配置多个模型端点（Ollama、DeepSeek，mimo-v2.5 后续可加其他）
  - 可在对话中随时**重置上下文**（清空历史，开始新话题）
  - 可选：自动创建/修改文件并保存。例如 claudian 里的“inline edit”功能：AI 生成修改建议后，插件直接替换原笔记内容，用户只需点“接受”。
- 暂时不做（第一版）：
  - 多模型同时比较
  - 对话历史持久化（刷新页面后历史丢失，但有重置按钮即可）

## 工作方式

- 修改前先阅读相关文件，不凭空猜测结构
- 涉及多个文件时，先给简短计划
- 不确定时先说明风险和假设
- 不要为了让代码跑通而注释掉错误
- 不要做与任务无关的重构

## 目录约定

- 源码目录：`src/`
  - `main.ts` – 插件入口
  - `settings.ts` – 设置界面与配置存储（API 端点、默认模型等）
  - `view.ts` – 侧边栏视图（对话 UI + 历史展示）
  - `conversation.ts` – 多轮对话状态管理（历史数组、添加上文、清空等）
  - `api/` – 各模型 API 封装（ollama.ts, deepseek.ts, xiaomi.ts），统一接收完整历史消息
- 测试目录：`tests/`（可选）
- 文档目录：`docs/`
- 临时实验目录：`scratch/`
- 禁止随意新增文件的位置：根目录、`.obsidian/`

## 开发命令

- 安装依赖：`npm install`
- 本地启动（开发模式）：`npm run dev`
- 运行测试：暂未配置，手动测试为主
- 代码检查：`npm run lint`（建议配置 ESLint + Prettier）

## 修改边界

- 不要硬编码 API Key（用户在设置中输入，本地 Ollama 不需要 Key）
- 不要删除用户的 Obsidian 配置或笔记
- 不要自动执行 git push 或部署
- 需要破坏性操作（如清除所有历史）时，需用户确认

## 验收标准

- 修改后说明改了什么
- 手动测试场景：
  - 在侧边栏连续提问 3 轮以上，AI 能正确引用前文
  - 点击“重置上下文”后，AI 不再记得之前的对话
  - 切换模型（Ollama ↔ DeepSeek）后，多轮对话状态不丢失
- 若测试失败，说明原因并指出修复方向

## 需要按需阅读的文档

- API规范：`docs/api.md`（以及docs目录下的其他.md文档）
- 数据库规范：无
- 前端规范：Obsidian 默认样式，避免引入额外 UI 库
- 发布流程：参考 [Obsidian 插件发布指南](https://docs.obsidian.md/Plugins/Releasing/Release+your+plugin+with+GitHub+Actions)
